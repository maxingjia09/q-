# 户外俱乐部网站数据库设计说明

## 数据库概述

- **数据库名称**: outdoor_club
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **存储引擎**: InnoDB
- **表数量**: 19张核心表
- **视图数量**: 3个
- **存储过程**: 2个
- **触发器**: 3个

---

## 表结构说明

### 1. users（用户表）

**用途**: 存储用户基本信息和账户状态

**字段说明**:
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 用户ID | 主键，自增 |
| username | VARCHAR(50) | 用户名 | 非空 |
| email | VARCHAR(100) | 邮箱 | 非空，唯一 |
| password | VARCHAR(255) | 密码（加密） | 非空 |
| phone | VARCHAR(20) | 手机号 | 可空 |
| avatar | VARCHAR(500) | 头像URL | 可空 |
| points | INT | 积分 | 默认0 |
| level | VARCHAR(20) | 会员等级 | 默认'普通会员' |
| is_admin | TINYINT(1) | 是否管理员 | 默认0 |
| status | TINYINT(1) | 状态 | 默认1（启用） |
| created_at | TIMESTAMP | 创建时间 | 自动生成 |
| updated_at | TIMESTAMP | 更新时间 | 自动更新 |
| last_login_at | TIMESTAMP | 最后登录时间 | 可空 |

**索引**:
- idx_email: 邮箱索引
- idx_username: 用户名索引
- idx_status: 状态索引

## 数据库设计特点

### 1. 规范化设计
- 遵循第三范式（3NF）
- 合理的外键约束
- 避免数据冗余

### 2. 索引优化
